# CRM 客户详情页

## 1. 页面目标

本页从原有“客户详情页”升级为“客户360执行视图”，在保留 CRM 后台骨架和客户档案卡的前提下，把中右内容区升级为围绕客户经营展开的协同工作台。

页面核心目标：

- 在客户详情页中统一承载“业绩目标 -> 任务 -> 多次进度提交 -> 订单联动 -> 结果分析”
- 让销售、跟单和运营在单页内完成目标创建、任务创建、进度推进和订单查看
- 让客户详情页可以看到与客户相关的一切核心内容，而不再只是静态资料页

## 2. 本次范围

### 2.1 本次包含

- 保留顶部品牌栏、左侧导航和左侧客户档案卡
- 左侧客户档案卡展示基础资料、客户经营摘要卡片与标签
- 客户基础信息下新增“收货地址”入口，点击后展示收货地址详情列表
- 收货地址列表支持新增、详情/编辑与删除，新增和编辑通过弹窗完成
- 客户基础信息下新增“联系人”入口，点击后展示联系人详情列表
- 联系人列表支持新增、详情/编辑与删除，新增和编辑通过弹窗完成
- 业绩目标横向滑动卡片区与状态筛选（筛选项展示对应目标数量，超过 2 个目标后滑动查看）
- “新建目标”弹窗
- 目标下任务列表
- “新建任务”弹窗
- 任务详情抽屉
- 任务进度多次提交
- 图片型附件缩略卡展示
- 当前客户订单联动弹窗
- 最新进度时间线（含进度提交与新增订单记录）
- 页面底部消费趋势和消费占比结果分析区

### 2.2 本次不包含

- 后端保存与真实接口
- 真实文件上传、图片管理、裁剪或预览大图
- 独立订单详情页
- 多人评论、审批流和权限控制

## 5. 数据说明

### 5.1 数据源优先级

1. 外部传入 `data`
2. 页面内置客户样例数据
3. `src/database/orders.json` 中的订单样例数据

### 5.2 外部数据项

页面支持以下数据项：

- `customer_detail`
- `goal_targets`
- `task_items`
- `task_progress_logs`
- `customer_orders`
- `customer_shipping_addresses`
- `customer_contacts`

### 5.3 关键数据结构

#### `customer_detail`

- `id`: 客户编号
- `display_name`: 客户简称
- `owner`: 归属人
- `follow_up_by`: 跟进人
- `customer_type`: 客户类型
- `annual_amount`: 近一年累计金额
- `last_follow_at`: 最近跟进时间
- `last_order_at`: 最近下单时间
- `receiver`: 默认收货人
- `phone`: 默认联系方式
- `shipping_address`: 默认收货地址

#### `goal_targets`

- `id`: 目标编号
- `customer_id`: 客户编号
- `title`: 目标名称
- `goal_type`: `金额目标` / `里程碑目标`
- `owner`: 负责人
- `start_date`: 开始日期
- `end_date`: 截止日期
- `status`: `进行中` / `有风险` / `已完成`
- `target_amount`: 目标金额，可选
- `current_amount`: 当前金额，可选
- `summary`: 目标说明
- `linked_order_ids`: 关联订单编号数组

#### `task_items`

- `id`: 任务编号
- `goal_id`: 所属目标编号
- `customer_id`: 客户编号
- `title`: 任务名称
- `task_type`: `新品推送` / `订单跟进` / `预售打板` / `客情维护` / `其他`
- `assignee`: 指定人
- `due_date`: 目标日期
- `status`: `未开始` / `进行中` / `有风险` / `已完成`
- `priority`: `高` / `中` / `低`
- `description`: 任务说明
- `linked_order_ids`: 关联订单编号数组
- `latest_progress_at`: 最近进度时间

#### `task_progress_logs`

- `id`: 进度编号
- `task_id`: 任务编号
- `customer_id`: 客户编号
- `submitted_by`: 提交人
- `submitted_at`: 提交时间
- `progress_percent`: 进度百分比
- `content`: 进度内容
- `status`: 提交后的任务状态
- `attachments`: 附件数组

#### `attachments`

- `id`: 附件编号
- `type`: 固定为 `image`
- `name`: 附件名称
- `preview_mode`: `warm` / `cool` / `neutral`

#### `customer_orders`

- `id`: 订单编号
- `customer_id`: 客户编号
- `customer_name`: 客户名称
- `order_time`: 下单时间
- `created_by`: 订单创建人，可选；无数据时以前目标负责人或当前跟进人兜底展示
- `product_name`: 商品名称
- `order_amount`: 订单金额
- `order_status`: 订单状态

#### `customer_shipping_addresses`

- `id`: 收货地址编号
- `customer_id`: 客户编号
- `receiver`: 收货人
- `mobile`: 联系方式
- `region`: 国家地区
- `detail_address`: 详细地址

#### `customer_contacts`

- `id`: 联系人编号
- `customer_id`: 客户编号
- `name`: 联系人
- `email`: 邮箱
- `position`: 职位
- `social_platform`: 社交平台
- `social_account`: 社交账号
- `phone_prefix`: 电话区号
- `phone`: 联系电话
- `birthday`: 生日
- `gender`: 性别
- `card_name`: 名片/头像文件名
- `remark`: 备注

## 6. 交互说明

### 6.1 页面级交互

- 页面默认选中第一个 `进行中` 的目标
- 点击目标卡后，任务区与右侧时间线同步切换到该目标上下文
- 左侧客户档案卡中的经营摘要区展示“最近进度 / 关联订单 / 近一年累计金额”三张卡片，其中“关联订单”卡片支持点击打开订单联动弹窗
- 点击“关联订单”卡片后，打开订单联动弹窗；弹窗默认展示当前客户订单，并优先排序和高亮当前目标关联订单
- 点击左侧客户基础信息入口中的“收货地址”后，打开收货地址详情列表弹窗
- 收货地址详情列表字段展示：收货人、联系方式、收货地址、操作；操作包含详情、编辑、删除
- 点击收货地址列表右上角“新增”后，打开新增收货地址弹窗
- 新增/编辑收货地址弹窗字段包含：收货人、收货人联系方式、国家地区、详细地址，并展示字符数提示
- 保存收货地址后即时刷新列表，删除后从当前列表移除
- 点击左侧客户基础信息入口中的“联系人”后，打开联系人详情列表弹窗
- 联系人详情列表字段展示：联系人、邮箱、职位、操作；操作包含详情、编辑、删除
- 点击联系人列表右上角“新增”后，打开新增联系人弹窗
- 新增/编辑联系人弹窗字段包含：联系人、邮箱、社交平台、联系电话、职位、生日、性别、名片/头像、备注，并展示字符数提示
- 保存联系人后即时刷新列表，删除后从当前列表移除
- 在订单联动弹窗中点击订单卡后，高亮与该订单关联的任务
- 右侧最新进度时间线展示当前目标下的进度提交和新增订单记录，订单记录需包含创建人、金额、时间和购买商品摘要
- 时间线头部不再展示“最新进度时间线”标题文案，仅保留右侧“跟进次数 / 下单笔数”横向统计，样式与“目标下任务”模块顶部统计一致
- “目标跟进总结”仅展示标题与摘要卡片，不再展示“已跟进 X 次，形成 X 笔关联订单”的单独描述文案；摘要卡片位于标题下方单独一行，并占满整行可用宽度，避免留下无意义空白
- 点击任务“查看”后，从右侧打开任务详情抽屉

### 6.2 目标交互

- 支持点击“新建目标”后打开弹窗表单，弹窗顶部仅保留标题与右上角关闭叉号
- 新建目标弹窗不展示目标类型字段，默认创建金额目标
- 目标卡不展示目标类型标签与顶部说明文案
- 状态筛选标签展示“全部 / 进行中 / 有风险 / 已完成”各自对应的目标数量
- 业绩目标卡片区不展示说明文案；超过 2 个目标时，右侧露出下一张卡片的一小部分边缘
- 目标卡片区左右两侧使用悬浮箭头平滑滑动到上一组/下一组内容；位于最左侧时隐藏左箭头，位于最右侧时隐藏右箭头
- “目标下任务”归属在业绩目标同一个容器内，作为当前选中目标的关联子区，不再独立成外层面板
- 滑动目标区按实际卡片高度收缩，不为未展示卡片预留空白，目标卡片区与“目标下任务”区保持紧凑关联
- 金额目标展示“任务完成数 + 金额达成金额”，不展示百分比文案
- 里程碑目标仅展示任务完成数，不展示“里程碑推进”文案

### 6.3 任务交互

- “新建任务”按钮固定放在“目标下任务”模块右上角，不再放在页面顶部操作区
- 点击“新建任务”后打开居中的弹窗表单，弹窗顶部展示标题与右上角关闭叉号
- 新建任务默认归属当前选中的业绩目标，保存后关闭弹窗并刷新当前目标下任务列表
- 新建任务弹窗展示字段：任务名称、指定人、目标日期、状态、优先级、任务说明
- 任务列表按字段展示：任务名称、状态、优先级、指定人、目标日期、关联订单数、最近更新、操作
- 任务列表中"目标日期"列：若任务状态不为"已完成"且目标日期已逾期，则在目标日期前展示红色"逾期"标签
- 任务详情抽屉内支持：
  - 不展示“任务详情抽屉”眉标，仅展示任务标题
  - 抽屉首屏直接展示所属目标、指定人、目标日期、任务状态、优先级、关联订单、最近更新、任务说明，不展示“基本信息”分组标题
  - 点击“关联订单”入口后，以弹窗方式展开订单绑定内容
  - 查看最新进度摘要的完整字段与附件，内容区不出现独立滚动条
  - 在抽屉底部点击按钮后，以弹窗方式提交任务进度
  - 查看完整进度时间线

### 6.4 进度提交流程

- 在任务详情抽屉底部按钮触发弹窗提交
- 支持输入文字、进度百分比、任务状态
- 支持挂载图片型附件缩略卡
- 提交后即时更新：
  - 任务最近更新时间
  - 任务状态
  - 右侧时间线
  - 客户最近跟进时间

## 8. 视觉规范

- 延续 CRM 客户列表页的浅色 B 端后台风格
- 主工作区以白色卡片和蓝色强调为主，突出“客户经营工作台”感
- 左侧客户档案卡中的经营摘要采用轻量卡片样式嵌入基础资料区，金额卡片通栏展示，最近进度与关联订单并排排列，避免额外形成突兀的顶部信息带
- 目标卡、目标下任务区、订单卡和任务抽屉需要有明确层级与选中态
- 风险态使用浅橙，完成态使用浅绿，进行中使用浅蓝
- 业绩目标卡片区使用横向滑动卡带，默认视口承载 2 张卡片并露出下一张边缘，不展示横向滚动条/进度条，仅通过悬浮箭头提示可继续查看
- 目标下任务区使用浅蓝内嵌容器与连接线表达“任务从属于当前选中业绩目标”的关系
- 订单联动内容收纳到居中弹窗中；左侧客户经营摘要中的“关联订单”卡片提供可点击状态和反馈
- 收货地址入口延续左侧客户档案卡浅蓝卡片风格，详情列表采用紧凑表格弹窗，新增/编辑弹窗采用用户截图中的小型表单布局
- 联系人入口延续左侧客户档案卡浅蓝卡片风格，详情列表采用紧凑表格弹窗，新增/编辑弹窗采用用户截图中的多字段小型表单布局
- 最新进度时间线展示区域加长，订单动态使用绿色节点区分于进度提交；头部首行展示目标跟进总结标题与右侧横向统计信息，第二行展示通栏摘要卡片
- 任务详情抽屉为高保真操作面板，不跳转新页面；抽屉正文采用整体滚动，任务关键信息与最新进度摘要区不使用内部滚动条

## 9. 验收标准

- 页面路径仍为 `/prototypes/crm-customer-detail`
- 从客户列表点击“详情”仍可按 `id` 正常进入
- 页面只展示当前客户相关目标、任务、进度和订单
- 默认样例数据同时包含金额目标与里程碑目标，并覆盖进行中、有风险、已完成三类目标状态
- 支持新建目标、新建任务、提交任务进度、绑定订单
- 新建任务入口位于“目标下任务”模块内，点击后以弹窗形式填写并保存任务
