# CRM 客户经营驾驶舱

## 1. 页面目标

本页新增统一的 CRM 客户经营驾驶舱，使用同一套页面框架承载“领导视角”和“业务员视角”两种使用方式。

页面核心目标：

- 让领导看到团队整体目标达成、业务员对比、重点客户维护和风险分布
- 让业务员看到自己负责或正在跟进的客户、个人目标达成、今日待跟进和风险任务
- 让页面支持从团队看板下钻到个人客户汇总，再继续跳转到客户详情页

## 2. 本次范围

### 2.1 本次包含

- 新增原型路径 `/prototypes/crm-dashboard`
- 支持 `view=leader` 与 `view=sales` 双视角切换
- 支持 `owner=姓名` 指定业务员个人视角
- 顶部经营总览区
- 目标与执行区
- 客户清单区
- 风险预警区
- 趋势分析区
- 领导视角业务员对比和下钻
- 业务员视角执行清单、目标达成与客户分层
- 点击客户跳转至现有 `/prototypes/crm-customer-detail?id=xxx`

### 2.2 本次不包含

- 后端真实接口与权限控制
- 多周期切换（年 / 季 / 周）
- 区域、国家、产品线等复杂切片分析
- 审批流、评论流和消息中心

## 3. 页面路径与参数

- 页面路径：`/prototypes/crm-dashboard`
- 视角参数：
  - `view=leader`：领导视角
  - `view=sales`：业务员视角
- 业务员参数：
  - `owner=陈敏`

示例：

- `/prototypes/crm-dashboard?view=leader`
- `/prototypes/crm-dashboard?view=sales&owner=陈敏`

## 5. 数据说明

### 5.1 原始输入数据

页面支持以下数据项：

- `customers`
- `goal_targets`
- `task_items`
- `task_progress_logs`
- `customer_orders`

### 5.2 关键原始字段

#### `customers`

- `id`: 客户编号
- `name` / `display_name`: 客户名称
- `owner`: 归属人
- `follow_up_by`: 跟进人
- `annual_amount`: 近一年累计金额
- `last_follow_at`: 最近跟进时间
- `last_order_at`: 最近下单时间
- `tags`: 标签
- `importance_level`: `A` / `B` / `C`，可选
- `next_follow_up_at`: 下次跟进时间，可选

#### `goal_targets`

- `id`: 目标编号
- `customer_id`: 客户编号
- `title`: 目标名称
- `owner`: 目标负责人
- `status`: `进行中` / `有风险` / `已完成`
- `target_amount`: 目标金额，可选
- `current_amount`: 当前金额，可选

#### `task_items`

- `id`: 任务编号
- `goal_id`: 所属目标编号
- `customer_id`: 客户编号
- `title`: 任务名称
- `assignee`: 指定人
- `due_date`: 截止日期
- `status`: `未开始` / `进行中` / `有风险` / `已完成`
- `priority`: `高` / `中` / `低`
- `latest_progress_at`: 最近进度时间

#### `task_progress_logs`

- `id`: 进度编号
- `task_id`: 任务编号
- `customer_id`: 客户编号
- `submitted_by`: 提交人
- `submitted_at`: 提交时间
- `progress_percent`: 进度百分比
- `status`: 提交后的任务状态

#### `customer_orders`

- `id`: 订单编号
- `customer_id`: 客户编号
- `customer_name`: 客户名称
- `order_time`: 下单时间
- `created_by`: 下单/创建人
- `product_name`: 商品名称
- `order_amount`: 订单金额
- `order_status`: 订单状态

### 5.3 聚合结果数据

页面内部会聚合生成以下结果结构，用于不同视角展示：

- `dashboard_summary`
- `dashboard_customer_rows`
- `dashboard_alerts`
- `salesperson_rows`

### 5.4 聚合规则

- 领导视角基于全量客户、目标、任务、进度和订单做团队汇总
- 业务员视角以 `owner` 为查询参数，统计“归属人为该业务员”或“跟进人为该业务员”的客户
- 若未提供 `importance_level`，则优先根据标签推断重点客户
- 若未提供 `next_follow_up_at`，则使用最近跟进时间与客户重要度兜底推断

## 6. 交互说明

### 6.1 页面级交互

- 默认进入 `leader` 视角
- 当 URL 为 `view=sales&owner=xxx` 时，切换到业务员视角并高亮当前业务员
- 领导视角点击业务员行后，可切换到对应业务员视角

### 6.2 客户与风险交互

- 点击客户名称或“查看客户”按钮后，跳转到 `/prototypes/crm-customer-detail?id=客户编号`
- 点击风险事项中的客户后，同样跳转到客户详情页
- 风险事项展示责任人、涉及客户、风险原因、建议动作与处理时限

### 6.3 业务员视角交互

- 展示“我的目标达成”“我的执行清单”“我的客户分层”
- 今日待跟进、即将到期、已逾期、最近有进展、最近形成订单 5 类清单分别展示
- 重点客户维护区展示重点客户当前状态和风险提醒
- 业务员视角的“我的目标达成”卡片结构对齐客户详情页“业绩目标”区域：展示状态、目标标题、目标摘要、进度条与底部元信息；同时补充展示目标所属客户名称
- 业务员视角的目标达成区支持与客户详情一致的目标状态筛选：`全部`、`进行中`、`有风险`、`已完成`
- 点击目标卡片后，下方展示该目标下任务列表；当前选中目标需有明确选中态
- “目标下任务”区提供与客户详情一致的“新建任务”按钮；点击后打开弹窗，包含任务名称、指定人、目标日期、状态、优先级、任务说明等字段，保存后立即出现在任务列表中
- 任务列表中的“查看”按钮打开任务详情抽屉，展示所属目标、客户、负责人、截止时间、任务状态、优先级、任务说明、最新进度摘要与完整进度时间线
- 业务员视角任务详情抽屉与客户详情页任务抽屉保持一致，底部固定展示“提交任务进度”按钮
- 点击“提交任务进度”后打开弹窗，字段包含提交人、进度%、任务状态与进度说明；提交后即时刷新任务状态、最近更新时间与时间线
- 客户经营表格中，客户级别列统一显示 `A级` / `B级` / `C级`，不额外展示错位的“有风险”“进行中”等标签，操作列统一为“进入客户”
- 目标状态列按客户目标聚合展示 `有风险X` 与 `进行中X`，同一客户可同时出现两个标签；若两者都没有则该列留空，且“未完成任务”显示为 `0 / 高优先 0`
- 客户清单区的四张摘要卡片支持点击筛选表格，再次点击当前卡片可取消筛选；筛选规则分别为：`A级客户=importance_level=A`、`正在推进=存在进行中或有风险目标`、`待补跟进=未完成任务为0`、`高优先未完结任务=高优先未完成任务大于0`

## 7. Axure API 说明

### 7.1 事件列表（eventList）

| 事件名称 | Payload 类型 | 触发时机 | 说明 |
|---------|-------------|---------|------|
| `on_switch_view` | `string` | 切换视角 | 返回当前视角与业务员 |
| `on_open_sales_view` | `string` | 领导视角下钻业务员 | 返回业务员姓名 |
| `on_change_owner_filter` | `string` | 切换业务员 | 返回当前业务员姓名 |
| `on_view_customer` | `string` | 点击客户 | 返回客户编号与来源视角 |

### 7.2 动作列表（actionList）

| 动作名称 | 参数 | 说明 |
|---------|------|------|
| `set_view` | `leader` / `sales` | 切换视角 |
| `set_owner` | 业务员姓名 | 切换业务员 |

### 7.3 变量列表（varList）

| 变量名称 | 类型 | 说明 |
|---------|-----|------|
| `current_view` | `string` | 当前视角 |
| `current_owner` | `string` | 当前业务员 |
| `customer_count` | `number` | 当前视角客户数 |
| `alert_count` | `number` | 当前视角风险数 |
| `key_customer_count` | `number` | 当前视角重点客户数 |
| `overdue_task_count` | `number` | 当前视角逾期任务数 |

## 8. 视觉规范

- 延续现有 CRM 客户列表页与客户详情页的浅色 B 端后台风格
- 以浅灰蓝背景、白色卡片、蓝色强调色作为主视觉
- 领导视角重点突出团队对比、风险提醒和重点客户维护
- 业务员视角重点突出“我今天要盯什么”和“我的目标做得怎么样”
- 风险态使用浅橙与红色提示，进行中使用浅蓝，完成态使用浅绿
- 卡片、表格、列表之间保持明显层级，避免堆砌成纯 BI 看板
- 桌面端以多列卡片+表格布局为主，移动端退化为单列堆叠
