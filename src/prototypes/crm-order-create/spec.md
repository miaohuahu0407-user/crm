# CRM 订单创建页

## 1. 业务与功能

### 1.1 核心目标

本页用于还原 CRM 后台中的“新建订单”场景，在维持高信息密度订单录入布局的同时，补充“按客户联动未完成任务”的新需求，让业务员在建单时可以直接把订单挂到当前客户的执行任务上，并在“客户”字段内通过单一控件完成类似省市联动的二级选择流程。

### 1.2 功能清单

- 订单基础信息录入：平台、平台单号、币种、客户、收货地址、业务员、跟进人、发货形式、时间、备注等字段
- 客户联动任务选择：点击“客户”单一控件后，先选择客户，再在同一弹出层内选择该客户未完成任务名称
- 任务记忆：同一客户再次被选中时，优先默认带出该客户上次选择过的任务
- 订单详情占位区：保留订单类型、商品明细表头和“请选择产品”入口
- 页面底部汇总与提交操作：展示金额汇总占位和取消/暂存/添加订单按钮

### 1.3 交互要点

- 切换客户时，自动更新收货地址、业务员、跟进人和关联任务候选集
- “客户”字段采用单一二级联动控件：先确认当前客户，再在同一控件的第二级列表中选择任务名称
- 若客户存在未完成任务：
  - 优先匹配该客户上次记忆的任务
  - 若记忆任务已完成或不存在，则保留任务二级下拉待用户手动选择
- 若客户不存在未完成任务：
  - 第二级任务列表展示“当前客户暂无未完成任务”空提示
- 用户手动切换任务后，立即写入本地记忆，用于下次选择同一客户时默认回填
- 点击“添加订单”时，对外抛出当前客户、任务、订单类型和平台单号等摘要数据

## 2. 信息架构

### 2.1 页面结构

```text
CRM 订单创建页
├─ 顶部品牌栏
├─ 左侧后台导航
├─ 顶部页签条
└─ 订单创建主卡片
   ├─ 页面标题：新增订单
   ├─ 基础信息区
   │  ├─ 常规订单字段
   │  └─ 客户字段内的任务二级联动选择
   ├─ 订单详情区
   │  ├─ 订单类型
   │  ├─ 商品明细表头
   │  └─ 请选择产品入口
   ├─ 底部汇总信息
   └─ 操作按钮区
```

### 2.2 数据来源

- 客户数据：优先读取 `data.customers`，无外部数据时使用页面内置样例
- 任务数据：优先读取 `data.task_items`，无外部数据时使用页面内置样例
- 任务记忆：使用浏览器本地 `localStorage`，按 `customer_id -> task_id` 存储最近一次选择

### 2.3 关键数据字段

#### `customers`

- `id`: 客户编号
- `name`: 客户名称
- `owner`: 业务员
- `follow_up_by`: 跟进人
- `shipping_addresses`: 收货地址列表

#### `task_items`

- `id`: 任务编号
- `customer_id`: 客户编号
- `title`: 任务名称
- `goal_title`: 所属目标名称
- `assignee`: 任务负责人
- `due_date`: 截止日期
- `status`: 任务状态，`已完成` 之外都视为未完成任务

#### 页面内部状态

- `selected_customer_id`: 当前客户编号
- `selected_task_id`: 当前任务编号
- `open_task_count`: 当前客户未完成任务数量
- `remembered_task_id`: 当前客户本地记忆的任务编号

## 3. 布局与响应

### 3.1 整体布局

- 布局模式：典型 CRM 后台双栏布局，左侧窄导航 + 右侧内容区
- 内容容器：顶部页签一行，下方主卡片纵向展开
- 表单布局：桌面端四列栅格；客户字段内部承载单一二级联动控件
- 订单表格：维持宽表头形式，窄屏下允许横向滚动

### 3.2 响应式适配

- 桌面端（>=1280px）：四列表单，完整展示后台信息密度
- 中屏（960px-1279px）：三列表单，订单表头横向滚动
- 平板与手机（<960px）：隐藏左侧导航，主表单降为两列或单列

## 4. 视觉规范

### 4.1 设计来源

设计规范来源：延续现有 CRM 后台原型的浅色 B 端风格，复用 `crm-customer-list` / `crm-customer-detail` 的品牌蓝、细边框、轻量表单和导航表达。

### 4.2 视觉要点

- 头部与左侧导航保持 CRM 系统一致的品牌识别
- 主体区域使用白底、浅灰边框和低阴影感，接近截图中的后台录入页
- “客户”字段使用类似级联选择的单一弹出控件，不额外展示任务数量提示、说明文案或任务摘要
- 激活页签和主操作按钮使用统一蓝色强调

### 4.3 状态定义

- 默认态：表单输入框白底灰边
- 禁用态：无未完成任务时任务下拉置灰
- 说明态：任务字段下方展示默认回填说明
- 激活态：当前页签和“添加订单”按钮使用主蓝色

## 5. Axure API

### 5.1 事件列表

| 事件名称 | Payload 类型 | 触发时机 | 说明 |
|---------|-------------|---------|------|
| `on_change_customer` | `string` | 切换客户 | 返回客户编号 |
| `on_change_task` | `string` | 切换关联任务 | 返回客户编号与任务编号 |
| `on_submit_order` | `string` | 点击添加订单 | 返回客户、任务、订单类型、平台单号等摘要 |

### 5.2 动作列表

| 动作名称 | Params 类型 | 参数说明 | 功能描述 |
|---------|------------|---------|---------|
| `reset_form` | `string` | 无 | 重置当前表单 |

### 5.3 变量列表

| 变量名称 | 类型 | 默认值 | 说明 |
|---------|-----|-------|------|
| `selected_customer_id` | `string` | 首个样例客户 | 当前客户编号 |
| `selected_task_id` | `string` | 自动计算 | 当前任务编号 |
| `open_task_count` | `number` | 自动计算 | 当前客户未完成任务数 |
| `remembered_task_id` | `string` | 本地记忆值或空 | 当前客户上次选择的任务编号 |

### 5.4 配置项列表

| 配置项名称 | 类型 | 默认值 | 说明 |
|----------|-----|-------|------|
| `operator_name` | `string` | `阿塔咪` | 右上角操作人名称 |

### 5.5 数据结构说明

页面支持通过 `data.customers` 和 `data.task_items` 覆盖默认样例数据；当外部未传值时，页面使用内置客户和任务样例完成演示。

## 6. 验收标准

- 页面路径为 `/prototypes/crm-order-create`
- 页面可正常展示 CRM 后台式订单创建界面
- 选择客户后，若该客户有未完成任务，则“关联任务”可选且只展示该客户未完成任务
- 同一客户再次被选择时，默认带出上次选择的任务
- 若记忆任务已完成、被删除或不属于当前客户，则任务名称二级下拉保留待选状态，不自动回退到首个任务
- 若客户没有未完成任务，任务字段禁用且展示空提示
- 页面通过 `typecheck` 和 `check-app-ready`
## 7. PRD 标注层

### 7.1 标注模块

- `page-context`：顶部导航、面包屑、页面标签与“新建订单”所在上下文
- `base-info`：订单基础信息主表单区域
- `customer-task-link`：客户字段内的客户与未完成任务级联选择
- `order-detail`：订单类型、商品表头和“选择产品”入口
- `submit-summary`：底部汇总信息与取消/暂存/提交操作

### 7.2 编辑与持久化

- 页面右侧提供固定标注面板，每个业务模块对应一个 badge。
- 标注内容通过富文本 `contenteditable` 区域直接编辑。
- 点击保存后，将当前标注状态写入浏览器 `localStorage`。
- 同时将已保存标注同步到页面内嵌 JSON 节点 `prd-annotation-export-state`。

### 7.3 导出行为

- `导出 Markdown` 会将整页当前标注内容下载为 `prd-page-annotation.md`。
- `导出发布版 HTML` 会导出带最新标注 JSON 状态的页面快照。
- 导出时必须使用当前编辑中的最新内容，而不只依赖上一次显式保存的文本。

### 7.4 补充验收

- 每个标注模块仅有一个可点击 badge。
- 点击 badge 后，右侧面板切换到对应模块需求说明。
- 保存后，刷新页面仍能从本地存储恢复已编辑标注。
- Markdown 和发布版 HTML 导出内容均包含最新标注文案。
