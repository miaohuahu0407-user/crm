# CRM 打板产品管理

## 1. 页面目标

本页用于管理 CRM 中的打板产品流程。页面核心目标：

- 用 `打板ID` 管理单条打板任务，而不是按产品挂规格
- 让业务、采购、跟单围绕同一条打板记录推进状态
- 支持客户打板与内部开发两类任务的差异化字段
- 支持废弃后从原单发起重打，并保留前后追溯关系
- 在打板确认完成后转正品，补录图库

## 5. 数据说明

### 5.1 数据源

1. 优先读取外部 `data.proofing_products`
2. 若未提供，则兼容读取旧结构 `data.presale_products`
3. 内置样例数据使用 `src/database/proofing_products.json`
4. 客户下拉默认读取 `src/database/orders.json`

### 5.2 关键数据结构

#### 打板产品

- `打板ID`
- `分类`
- `图片`
- `需求描述`
- `期望完成时间`
- `当前状态`
- `打板类型`：`内部开发` / `客户打板`
- `客户ID` / `客户名称` / `打板费用`
- `供应商ID` / `供应商名称` / `供应商打板价` / `供应商确认时间`
- `样式图片` / `样式确认时间`
- `颜色图片` / `颜色确认时间`
- `快递公司` / `运单号` / `快递费`
- `收货时间`
- `收货操作人`
- `寄送客户时间`
- `图库`
- `正品产品ID`（兼容历史数据，可为空）
- `转正时间`
- `原打板ID`
- `重打次数`
- `后置状态`：`未转正品` / `已转正品` / `已废弃`。该字段作为全流程独立的正品状态使用，默认值为 `未转正品`；无论打板记录当前处于 `待打板`、`打板中`、`待确认颜色`、`待收货` 还是 `已收货`，都通过此字段标记是否已转正或已废弃
- `寄送状态`：界面展示字段，不单独存储；`客户打板` 在 `已收货` 后可寄送给客户，未寄送显示 `未寄送`，寄送成功后依据 `寄送客户时间` 标记为 `已寄送`

### 5.3 旧结构兼容

- 旧 `presale_products` 中的每条规格会被折算为 1 条打板记录
- 旧状态兼容映射：
  - `待打板` → `待打板`
  - `打板中` → `打板中`
  - `已打板` → `打板中`
  - `已确认` → `已收货`
  - `已转正品` → `已转正品`
  - `已废弃` → `已废弃`

### 5.4 图片资源

内置样例数据的 `图片` 与 `图库` 字段为空，避免外部 URL 在国内网络环境下的加载失败问题。

无图片记录使用内置 SVG 兜底图 `buildCategoryFallbackImage`，按分类生成对应首饰风格的矢量图案：
- **皇冠**：金色渐变（#d97706 → #fbbf24）+ 皇冠轮廓 + 宝石点缀
- **项链**：蓝色渐变（#2563eb → #60a5fa）+ 项链弧线 + 菱形吊坠
- **耳环**：玫瑰金渐变（#db2777 → #f472b6）+ 耳钉 + 水滴形耳坠
- **手链**：青绿渐变（#059669 → #34d399）+ 手镯椭圆轮廓
- **其他**：紫色渐变（#7c3aed → #a78bfa）+ 钻石切面

所有 `<img>` 标签均绑定 `onError={handleImgError}`，图片加载失败时自动回退到 SVG 兜底图。

## 6. 交互说明

### 6.1 主视图

- 页面仅保留 `打板产品管理` 单视图，展示全部打板任务
- `待打板` 记录内直接执行 `供应商确认`，确认后状态改为 `打板中`

### 6.2 分类侧栏

- 点击分类项切换筛选
- 侧栏展示当前列表下的分类数量
- `全部` 始终置顶

### 6.3 打板产品列表

- 主识别信息为 `打板ID + 图片`，其中 `打板ID` 列只显示 ID 本身，不展示辅助小标题或副文案
- 辅助字段展示：分类、打板类型、客户、期望完成时间、状态、供应商/打板价
- 支持按 `打板ID`、需求描述、客户、供应商搜索
- 支持按状态筛选，状态按钮展示当前分类下的数量
- 支持通过下拉选项按 `正品状态` 筛选
- `内部开发` 类型在列表客户列显示为空
- 列表操作按钮保持同一行并列展示，不做上下堆叠
- 列表操作区按钮仅展示文字，不展示图标
- 点击图片或“详情”打开详情抽屉
- `待打板` 记录在列表操作区展示 `供应商确认` 按钮
- `打板中` 记录在列表操作区展示 `确认样式` 按钮，点击后弹出样式确认弹窗
- 全流程都展示 `正品状态`：`未转正品` / `已转正品` / `已废弃`
- `待打板`、`打板中`、`待确认颜色` 阶段在列表和详情中展示：
  - 打板状态：当前生命周期状态
  - 正品状态：`未转正品` / `已转正品` / `已废弃`
- `待收货` 与 `已收货` 阶段在列表和详情中展示：
  - 收货状态：`未收货` / `已收货`
  - 正品状态：`未转正品` / `已转正品` / `已废弃`
- 顶部筛选栏新增 `正品状态` 下拉：`全部` / `未转正品` / `已转正品` / `已废弃`
- 收货状态为 `未收货` 时，列表操作区展示 `标记已收货`
- 正品状态为 `未转正品` 时，列表操作区展示 `转正品`，适用于 `待打板`、`打板中`、`待确认颜色`、`待收货`、`已收货`
- `已收货` 且 `客户打板` 类型在未寄送客户前，列表操作区展示 `寄送给客户`
- 寄送成功后，列表状态区与详情抽屉展示 `已寄送` 标记
- 正品状态为 `已废弃` 的记录展示 `发起重打` 按钮
- `供应商确认` 后写入供应商、供应商打板价与确认时间

### 6.5 新建打板产品

- 必填字段：分类、打板类型、图片、需求描述、期望完成时间
- `客户打板`：
  - 必须单选客户
  - 必须填写打板费用
- `内部开发`：
  - 不展示客户与打板费用
- 创建成功默认状态为 `待打板`

### 6.6 详情抽屉

抽屉采用单列纵向流布局，仅保留状态时间轴与底部操作按钮区；底部操作按钮区固定在抽屉底边，不随时间轴内容滚动：

**状态时间轴**：按当前生命周期阶段动态展示，仅展示已到达及当前阶段，未到达的阶段不展示；每个节点下补充对应模块信息，含 `供应商确认` 与 `确认样式` 操作按钮；时间轴内不展示 `已转正品` 节点。
- `待打板` 节点摘要：已确认时显示 `供应商已确认 + 确认时间`，未确认时显示 `待确认供应商与打板价`；详情与操作按钮仅在阶段已完成后展示
- `打板中` 节点仅保留样式图片，不展示确认状态、确认时间与辅助提示；图片尺寸与抽屉头部打板图片一致，点击可放大查看
- `待确认颜色` 节点仅保留颜色图片，不展示确认状态、确认时间与辅助提示；图片尺寸与抽屉头部打板图片一致，点击可放大查看
- `待收货` 节点摘要：已收货时显示 `{操作人} 已收货 {收货时间}`，未收货时按原逻辑展示；不展示操作人与操作时间详情卡片
- `已收货` 节点不展示摘要与详情卡片

底部操作按钮区按状态展示：供应商确认、确认样式、上传颜色图、标记已收货、寄送客户、转正品、废弃、发起重打；操作区始终停靠在抽屉底部。

其他规则：
- 保留新旧单关系：`原打板ID`、`重打次数`
- 抽屉头部左侧展示打板图片，点击可放大
- 抽屉头部展示打板编号、生命周期状态、类型标签；已寄送时同步展示寄送状态
- 抽屉头部展示真实需求描述，不再使用固定占位文案
- 抽屉头部在标题行与需求描述之间以紧凑行内形式展示基础元信息：`创建人 / 创建时间` 与 `期望完成日期`，不使用卡片
- `客户打板` 在需求描述下方以紧凑行内形式展示 `客户与物流`，格式为 `客户：xxx | 打板费：xxx | 快递：xxx | 运单：xxx | 运费：xxx`
- 当正品状态为 `已转正品` 时，在需求描述下方以紧凑行内形式展示 `转正信息`，格式为 `编号：xxx | 时间：xxx | 图库：...`

### 6.7 状态推进

- `待打板`：点击 `供应商确认`，填写供应商与供应商打板价后进入 `打板中`
- `待打板` / `打板中` / `待确认颜色`：正品状态默认显示为 `未转正品`，在未转正时均可直接执行“转正品”
- `打板中`：点击“确认样式”弹出样式确认弹窗，样式图片非必填，提交后进入 `待确认颜色`
- `待确认颜色`：点击“确认颜色”上传颜色图片，完成后进入 `待收货`
- `待收货`：点击“标记已收货”进入 `已收货`，并记录确认收货的操作人、操作时间；正品状态为 `未转正品` 时也可直接执行“转正品”或“废弃”
- `已收货`：收货状态显示为 `已收货`；正品状态可继续保持 `未转正品`、变更为 `已转正品` 或 `已废弃`
- `已收货` 且 `客户打板` 类型在未寄送客户前展示"寄送给客户"按钮，填写快递公司、运单号、快递费后确认寄送，记录仍留在 `已收货` 状态，并显示 `已寄送`
- 转正品或废弃后，当前打板主状态保持不变，通过 `后置状态` 标记为 `已转正品` 或 `已废弃`
- `已废弃`：可发起重打

### 6.8 客户打板寄送

- `客户打板` 类型在 `已收货` 状态且未寄送客户前，展示"寄送给客户"按钮
- 点击后弹出寄送弹窗，必须填写：
  - 快递公司
  - 运单号
  - 快递费
- 确认寄送后记录 `寄送客户时间`，记录仍留在 `已收货` 状态，并标记为 `已寄送`

### 6.9 废弃与重打

- `待收货` 与 `已收货` 状态且未设置后置状态时可废弃
- 废弃后主状态保持不变，后置状态标记为 `已废弃`
- 后置状态为 `已废弃` 的记录支持“发起重打”
- 重打会生成新的打板记录：
  - 新记录状态回到 `待打板`
  - 新记录保留原始分类、图片、需求描述、客户信息
  - 原记录后置状态保留 `已废弃`
  - 新旧记录可追溯

### 6.10 转正品

- 全流程在 `未转正品` 状态时均允许转正
- 转正时必须：
  - 上传图库
- 转正后主状态保持不变，后置状态标记为 `已转正品`

## 9. 验收标准

- 页面路径为 `/prototypes/crm-presale-product`
- 页面仅保留 `打板产品管理` 单视图
- 列表不再出现规格字段与规格操作
- 新建 `内部开发` 不要求客户与打板费用
- 新建 `客户打板` 必须单选客户并填写打板费用
- `待打板` 记录可直接执行 `供应商确认`，确认后状态变为 `打板中`
- 状态顺序为：
  - `待打板 → 打板中 → 待确认颜色 → 待收货 → 已收货`
- `打板中` 记录点击“确认样式”会弹出弹窗，样式图片为选填
- 确认颜色时可上传颜色图片，确认后进入 `待收货`
- 详情抽屉头部展示真实需求描述；所有记录都展示创建信息与期望完成日期，`客户打板` 额外展示客户名称、打板费、快递公司、运单号、运费
- 已转正品记录的详情抽屉头部额外展示正品编号、图库与转正时间
- 详情时间轴按当前生命周期阶段动态展示，仅展示已到达及当前阶段，不展示未到达的阶段与“已转正品”节点；每个节点需补充对应业务模块信息
- `待打板`、`打板中`、`待确认颜色` 阶段必须同时展示“打板状态”和“正品状态”
- `待收货` 与 `已收货` 阶段必须同时展示“收货状态”和“正品状态”
- 收货状态为 `未收货` 时展示 `标记已收货` 按钮；收货状态为 `已收货` 且 `客户打板` 未寄送时展示 `寄送给客户`
- 正品状态为 `未转正品` 时，全流程展示 `转正品`；正品状态为 `已废弃` 时展示 `发起重打`
- 后置状态为 `已废弃` 的记录可发起重打，新旧记录存在回链
- `待打板`、`打板中`、`待确认颜色`、`待收货`、`已收货` 在正品状态为 `未转正品` 时均可转正品
- `待收货` 与 `已收货` 状态且未设置后置状态时仍可废弃
- 转正品或废弃后，主状态保持不变，通过后置状态标记处理结果
- `已收货` 且 `客户打板` 类型在未寄送客户前展示"寄送给客户"按钮
- 寄送客户时需填写快递公司、运单号、快递费，确认后记录仍留在 `已收货` 状态，并显示 `已寄送`
- 页面通过 `typecheck` 与 `check-app-ready`
